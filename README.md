# Google Smartphone Decimeter Challenge 2022

GNSS positioning pipeline for the [Google Smartphone Decimeter Challenge 2022](https://www.kaggle.com/competitions/smartphone-decimeter-2022) on Kaggle. Predicts phone position from raw GNSS measurements to decimeter-level accuracy.

**Best score:** 2.466m private / 2.524m public
**Position:**  56/573 on Public leaderboard
*Note: Official Submission to kaggle public/private leaderboard is not as Competition has already closed*

## Pipeline

```
Raw GNSS -> Hatch Filter (N=1000) -> Cauchy WLS -> Kalman Smoother (sigma=17)
         -> LightGBM Correction (46 features, 5-seed ensemble) -> SavGol Smoothing
```

## Project Structure

```
01_submission.ipynb          # Generate and submit final predictions
02_training_experiments.ipynb # Model training, parameter sweeps, experiment log
src/
  gnss_solver.py             # Hatch filter, WLS solver, Kalman smoother
  ml_correction.py           # LightGBM per-epoch error correction
  data_loader.py             # GNSS/IMU data parsing
  post_processing.py         # Trajectory smoothing (SavGol, median filter)
  feature_engineering.py     # Per-satellite ML feature extraction
  main.py                    # Pipeline orchestration and CLI
models/                      # Trained LightGBM models (5-seed ensemble)
tests/                       # Unit tests
```

## Quick Start

```bash
pip install -r requirements.txt
```

**Generate a submission:** Open and run `01_submission.ipynb` end-to-end. Requires `kaggle_dataset/` with competition data and trained models in `models/`.

**Retrain models:** Open and run `02_training_experiments.ipynb`. This regenerates training data (~60 min), trains LightGBM with cross-validation, and saves the 5-seed ensemble.

**Run tests:**
```bash
pytest tests/
```

## Data

Place the competition dataset under `kaggle_dataset/` with `train/` and `test/` subdirectories. Training data CSV (`ml_training_data.csv`) is generated by `02_training_experiments.ipynb`.

## Attribution

The core GNSS solver is adapted from [Taro Suzuki's](https://www.kaggle.com/taroz1461) public Kaggle notebook ([Carrier Smoothing + Robust WLS + Kalman Smoother](https://www.kaggle.com/code/taroz1461/carrier-smoothing-robust-wls-kalman-smoother)). Suzuki placed 1st in the competition; his winning method used factor graph optimization (GTSAM), but this project adapts the simpler WLS+Kalman approach from his public notebook.

Reference: Suzuki, T. (2023). "Precise Position Estimation Using Smartphone Raw GNSS Data Based on Two-Step Optimization." *Sensors*, 23(3), 1205.

Phone-specific bias modelling was inspired by [J.B.O. Mitchell's](https://www.kaggle.com/jbomitchell) phone-specific bias correction and tectonic correction notebooks, which built on [saitodevel01's](https://www.kaggle.com/code/saitodevel01/gsdc-bias-eda) lever arm bias analysis from the 2021 competition.

All code and implementation in this project is original. See `src/gnss_solver.py` and `src/ml_correction.py` headers for detailed attribution.
